  • What: RESTful API for an e-commerce product catalog
  • Purpose: Serve product data, categories, reviews; support admin CRUD via JWT
  • Architecture: Django 5 + Django REST Framework
  • Base URL: /api/

Data Model Rationale

  • Category: Hierarchy & filtering; slug for SEO; PROTECT on delete to prevent orphaned products
  • Product: Decimal price & validators; indexes on price, created_at, is_available for fast queries
  • Review: unique_together (product, name); 1–5 rating; CASCADE so reviews go with product
  • Performance: Indexes on slug, category+price, is_available; select_related/prefetch_related to avoid N+1

API Features

  • Filtering: category_slug, min_price, max_price, is_available
  • Search: name, description, category__title
  • Ordering: name, price, stock_quantity, created_at
  • Pagination: Limit-offset (default 20 per page)
  • Auth: JWT (Simple JWT); IsAuthenticatedOrReadOnly for catalog
  • Docs: Swagger at /api/docs/, ReDoc at /api/docs/redoc/, schema at /api/docs.json
